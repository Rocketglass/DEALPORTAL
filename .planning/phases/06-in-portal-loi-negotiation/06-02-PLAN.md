    Left column (main):
    - LOI header card: property name, suite, tenant/landlord/broker names, sent date, status badge
    - Info banner: "Review each section and accept, counter, or reject." (blue-50 background)
    - Section cards (reuse the visual structure from loi-sections-panel.tsx):

      - Agreed value (if exists, in green)

      - Counter textarea (shown when counter selected)
      - Reject reason textarea (shown when reject selected)
      - **Per-section submit button**: "Submit Response" — submits ONE section at a time (not batch). This is better for multi-party negotiation where parties respond section by section. POST to `/api/lois/${loiId}/respond` with `{ responses: [{ sectionId, action, value?, note?, updatedAt }] }`. On success, refetch review-data to update the UI.
      - Expandable negotiation history timeline (same visual as loi-sections-panel.tsx):
        - Timeline entries with dot color by role: broker=#1e40af, landlord=amber-500, tenant=emerald-500
        - Three-color legend: Broker (blue), Landlord (amber), Tenant (green)
        - Each entry shows: role name, action badge, value if counter, note if present, timestamp

    Right column (sidebar, sticky):
    - Negotiation Progress card: progress bar + breakdown by status (accepted/countered/proposed/rejected counts)
    - Key dates card: created, sent, expires, agreed, version

    **State management:**
    - `sections` state refreshed after each submit
    - `submittingSection` tracks which section ID is currently submitting (show spinner on that section's button)
    - After successful submit, show a brief success toast or inline "Response recorded" message that fades

    - Three-party timeline legend (Broker=blue dot, Landlord=amber dot, Tenant=green dot) — extend the existing two-party legend
    - Responsive: full sidebar on lg, stacked on mobile
