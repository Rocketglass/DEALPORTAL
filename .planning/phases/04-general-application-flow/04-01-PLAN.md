Create migration `015_application_properties.sql` with two tables:

1. **application_properties** junction table:
```sql
CREATE TABLE application_properties (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  application_id UUID NOT NULL REFERENCES applications(id) ON DELETE CASCADE,
  property_id UUID NOT NULL REFERENCES properties(id) ON DELETE CASCADE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(application_id, property_id)
);
CREATE INDEX idx_app_props_application ON application_properties(application_id);
CREATE INDEX idx_app_props_property ON application_properties(property_id);
```

2. **application_drafts** table for server-side save:
```sql
CREATE TABLE application_drafts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  form_data JSONB NOT NULL DEFAULT '{}',
  selected_property_ids UUID[] DEFAULT '{}',
  current_step INTEGER NOT NULL DEFAULT 1,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id)  -- one draft per user
);
CREATE INDEX idx_app_drafts_user ON application_drafts(user_id);
```

- `application_properties`: broker_all (is_broker_or_admin()), tenant select via contact email match through applications table
- `application_drafts`: user can select/insert/update/delete their own row (`user_id = (SELECT id FROM users WHERE auth_provider_id = auth.uid())`), broker_all for support

4. Enable RLS on both tables. Add updated_at trigger on application_drafts.

- Auth required — use `requireAuthForApi()` (any authenticated user can save a draft, not just tenants, because a user filling out the form may not have role=tenant yet)
- **GET**: Fetch the user's draft from application_drafts where user_id matches the auth user's users.id. Return `{ draft: { form_data, selected_property_ids, current_step } | null }`.

- Use service client for DB operations (user may not have RLS-visible access to their own draft row during initial creation race window).
- Look up the user's `users.id` via `auth_provider_id = supabase.auth.getUser().id` to match against application_drafts.user_id.

Note: The middleware already allows `/api/applications` as a public API route, but `/api/applications/drafts` is NOT in the public list, so it correctly requires auth via middleware. No middleware changes needed.
